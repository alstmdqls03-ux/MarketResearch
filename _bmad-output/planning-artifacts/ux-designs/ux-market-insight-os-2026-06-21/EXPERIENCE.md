# Market Insight OS — Experience Spine

> 투자 판단 디버거. 매수 직전 30초와 한 달 뒤 복기 사이의 피드백 루프를 압축한다. 시각 정체성은 `DESIGN.md`, 본 문서는 경험 계약이다. FR/NFR 상세는 위 sources를 참조한다(중복 기술하지 않음).

## Foundation

폼팩터는 **하나의 모바일 우선 반응형 웹앱**을 네이티브 **WebView 셸**로 감싸 앱으로 배포하는 형태다. 웹과 앱이 **동일 코드베이스** — 웹 품질이 곧 앱 품질이다. 데스크톱은 같은 코드에서 대시보드를 **멀티 컬럼**으로 확장한다. 실시간 시세는 없다(EOD/지연) → 폴링/수동 새로고침. PWA·오프라인은 MVP 제외(Future).

UI 시스템은 **shadcn/ui + Tailwind**. 컴포넌트 라이브러리가 대부분의 일을 하고, 브랜드 규율은 "기본을 존중하되 브랜드 레이어만 오버라이드"다. 시각 정체성·토큰은 `DESIGN.md`가 소유하며, 본 문서는 토큰을 `{path.to.token}`으로 이름 참조한다(예: `{colors.up}`, `{components.research-card}`).

> **Conventions — 스파인 우선:** 두 스파인(`DESIGN.md` + 본 문서)이 목업/와이어프레임과 충돌 시 **스파인이 우선한다.** 목업은 합성 레퍼런스일 뿐 계약이 아니다.

## Voice and Tone

마이크로카피. 브랜드 보이스·미적 태도는 `DESIGN.md`.

핵심 원칙: **거울/호기심**(성적표·잔소리 아님), **데이터 인용 필수**(뻔한 공자말씀 0건), **추천·예측 금지**(종목의 미래·적합성 절대 언급 안 함 — 규제 방패이자 제품 원칙). **네이밍 가드:** alpha / picks / signals / journal(오염어) 회피 → "판단 디버거 / 의사결정 리뷰" 언어 사용.

## Component Patterns

행위. 시각 스펙은 `DESIGN.md.Components`.

| 컴포넌트 | 사용처 | 행위 규칙 |
|---|---|---|
| 리서치 카드 `{components.research-card}` | 관심종목, 온보딩 | 종목당 1개. 상승논리/하락리스크 분리 표시. 동일 종목 재요청은 캐시로 즉시(NFR-P3). 하단 비강제 넛지로 30초 체크 진입. |
| 칩 `{components.chip}` | 30초/매도 체크 | 탭으로 단일/다중 선택, 즉시 반영(<100ms). **선택 상태는 `aria-pressed`(또는 토글 그룹 `aria-checked`)로 프로그램적 노출** — 색만으로 선택을 전달하지 않는다. **"기타(직접)" 칩은 타이핑 입력으로 전환**(거짓 기록 방지, NFR-U4); 전환 시 입력 필드로 포커스 이동, Esc 취소 시 원래 칩으로 포커스 복귀. |
| 점진 공개(감정 탭→질문→필드) | 30초/매도 체크 | 감정 탭 → "계획인가 감정인가" 질문 출현 → 이유·리스크·기간 필드 펼침. **새로 드러나는 영역은 SR에 announce**(`aria-live="polite"` 또는 펼침 직후 첫 요소로 포커스 이동), 논리적 포커스 순서 유지. 타이핑-선택 경로 전체가 키보드+SR로 완전 운용 가능해야 한다. |
| Primary 버튼 `{components.button-primary}` | 카드/체크 CTA | 화면당 1개 주 행동. 탭 후 비동기면 진행 표시로 전환. |
| as-of 배지 `{components.asof-badge}` | 시세 노출 전 표면 | 모든 시세에 신선도 시각 동반(FR32). 임계 초과 시 가시적 degrade. |
| 면책 `{components.disclaimer}` | 모든 AI 출력 | 카드·통찰·복기마다 per-output 1줄(FR34). |
| Insight Score (거울) `{components.score-mirror}` | Score, 대시보드 | 채점이 아니라 패턴 문장 + 추세. "비추기"가 행위의 본질. |
| 비동기/재시도 `{components.async-state}` | 리서치·복기 생성 | UI 차단 금지, 진행 표시. 실패 시 가시적 알림 + 재시도(FR42). |

## State Patterns

| 상태 | 표면 | 처리 |
|---|---|---|
| 빈상태/온보딩 | 온보딩, 관심종목 | "관심종목 1개만." 입력 즉시 리서치 카드. 결정 0이어도 빈손 이탈 방지. |
| 로딩/비동기 AI | 리서치 카드, 복기 | shadcn `Skeleton` + 진행 표시. 캐시 히트는 즉시. UI 비차단(NFR-P3). |
| 오류(가시적) | AI 생성, 시세 | **조용히 넘어가지 않음.** 가시적 오류 + 재시도(FR33, FR42). 누락 지표는 경고 병기. |
| as-of/stale | 시세 노출 표면 | as-of 시각 표기. 신선도 임계 초과 시 앰버 degrade(NFR-R2). 복기는 *사용자가 본 정보로만* 평가(공정성, FR22). |
| append-only/버전 | 결정 이력 | 기록된 결정은 수정 불가, 변경은 버전으로 누적(FR15). |
| 원자 결정+스냅샷 | 매수/매도 체크 | 결정 쓰기 + 스냅샷 = 단일 트랜잭션. 스냅샷 실패 시 결정도 롤백 — **고아 결정 0**(FR12, NFR-R1). |
| 입력 거부/미지원 종목 | 관심종목, 온보딩 | 유효하지 않거나 미지원 티커 입력 시 **조용히 무시하지 않고 가시적으로 거부 + 안내 메시지**(예: "지원하지 않는 종목이에요. 미국 티커를 확인해 주세요"), 재입력 유도(FR43). |
| 운영자/파이프라인 상태 | 운영자(최소) 화면 | 파이프라인 실패를 **운영자에게 가시적으로 표시**(실패 큐·재시도 진행·복구), 재시도 액션 제공(FR37). 사용자측 "데이터 보강 중"과 분리. 상세는 별도 내부 스펙. |

> **데이터 함의 (DB GUARDRAIL · 사용자 지시 · 다운스트림 아키텍처가 반드시 따를 것):** 테이블 최소화 + JSONB 우선. 하나의 투자 판단 근거(이유·최대 리스크·투자기간·감정·목표가·손절)와 결정 시점 스냅샷(가격·지표·as-of)은 **쪼개지 말고 결정 행의 JSONB로 통째 저장**한다. 본 UX 플로우의 어떤 입력도 별도 테이블 분리를 함의하지 않는다. 모델: **단일 `decisions` 테이블 · (user, ticker) 1:N · 매도→매수 self-reference(FR40) · JSONB blob의 append-only 버전(FR15).** 30초 체크의 다중 칩 입력은 여러 행/테이블이 아니라 한 결정 행의 JSONB 필드로 모인다.

## Interaction Primitives

- **점진 공개(progressive disclosure):** 30초 체크는 *감정 한 번 탭 → "지금 계획인가, 감정인가?" → 나머지(이유·리스크·기간·목표가·손절)는 선택*. 첫 화면을 가볍게 시작해 단계로 펼친다.
- **칩 입력 + "기타":** 모든 필수 입력은 탭/선택으로 완료. "기타(직접)"는 타이핑 탈출구이자 거짓 기록 방지 장치(NFR-U4).
- **저마찰(타이핑 선택) ↔ 의도된 마찰 균형:** 핵심 루프는 탭/스크롤 우선, 타이핑은 항상 선택(NFR-U1). 단 **30초는 0으로 최적화하지 않는다** — "정직한 생각 한 박자"를 강제하는 과속방지턱(NFR-U2). 체크 입력 반응은 <100ms(NFR-P1)지만 흐름 자체를 0초로 줄이지 않는다.
- **비강제 넛지(막지 않고 직면):** 충동 감정(FOMO/공포) 입력 시 마이크로 개입("계획인가 감정인가")을 제시하되 **행동을 막지 않는다**. 매도하더라도 그 결정은 기록되어 다음 복기 대상이 된다(FR14).
- **비동기 생성 + 진행 + 재시도:** 리서치 카드·복기는 수 초 소요 → 비동기 + 진행 표시, UI 비차단, 실패 시 재시도(NFR-P3, FR42).
- **금지:** 실시간 틱 깜빡임, 등락 깜빡이 애니메이션, 강제 모달 게이트, 행동 차단형 경고.

## Accessibility Floor

행위. 시각 대비는 `DESIGN.md`(shadcn AA 기본 + 브랜드 오버라이드 검증).

- **WCAG 2.1 AA 지향**(NFR-AC1) — 대비·키보드 조작·스크린리더 레이블 기본 충족.
- 탭 타깃 **≥44px**, 핵심 액션은 **엄지 도달 영역**(NFR-U3).
- 30초 체크 입력 **반응 <100ms**(즉각 느낌, NFR-P1).
- **색만으로 등락 전달 금지** — 방향 색에 부호/화살표 글리프(▲▼, +/−) 병기(색맹 안전, `DESIGN.md` Do/Don't와 정합).
- 스크린리더: 등락은 "상승/하락 + 수치"로 읽고, as-of/면책은 출력에 함께 announce.
- **칩 선택 상태는 `aria-pressed`/`aria-checked`로 노출**(색 단독 금지). **점진 공개로 새로 드러나는 콘텐츠는 `aria-live` 또는 포커스 이동으로 announce**하고 논리적 포커스 순서를 보장한다. 타이핑-선택 경로 전체가 키보드+SR로 완전 운용 가능해야 한다.
- 비동기 오류/재시도는 `role="alert"`/`aria-live="assertive"`로 announce, 진행 표시는 `aria-busy`로 노출.
- **30초 "과속방지턱"은 강제 시간제한이 아니라 self-paced**(사용자가 멈춰 생각하게 하는 의도적 마찰일 뿐, 카운트다운·자동 진행·세션 만료 없음 → WCAG 2.2.1 Timing Adjustable 비위반).
- Tab 순서는 읽기 순서, `Esc`는 최상단 모달/팝오버를 닫는다.

## Key Flows

### Flow J0 — 온보딩: 빈손으로 안 나감 (신규 사용자)

1. 가입 직후 — 관심종목 0, 결정 0.
2. 앱은 **단 하나만** 요청한다: "관심종목 1개만."
3. 입력 즉시 리서치 카드(상승논리|하락리스크 분리)가 떠 즉각 가치를 준다.
4. "이 종목, 사보신 적 있나요?" → 있으면 과거 1건을 30초로 소급 기록(스냅샷은 '기록 시점'으로 정직하게 표시, FR17).
5. **클라이맥스:** 첫 세션을 빈손이 아니라 **카드 1개(+선택적 기록 1건)를 들고** 나간다. 빈 대시보드 이탈을 정면 돌파.

### Flow 1 — 민수(Primary) 성공 루프: 첫 결정에서 통찰을 받다

1. 민수가 NVDA를 실적 기대에 매수하려 한다(평소엔 그냥 샀다).
2. 리서치 카드에서 *상승논리 | 하락리스크*를 분리해 본다.
3. 매수 직전 30초 체크 — 감정 한 번 탭 → "지금 계획인가, 감정인가?" → 나머지는 선택(점진 공개, 타이핑 없이). 감정에 'FOMO'.
4. 기록과 동시에 결정 시점 스냅샷(민수가 그 순간 본 정보 집합 + as-of)이 **원자 저장**된다.
5. **클라이맥스:** *결정 #1인데도* 앱이 말한다 — "12% 급등 후 매수네요. 적으신 최대 리스크는 '고평가', 다음 실적일 D-9. 이 진입가가 그 리스크를 존중하나요?" 민수가 처음으로 멈칫한다.
6. 한 달 뒤 자동 복기: "결과는 좋았으나 horizon은 '1주'였고 FOMO 진입 — 프로세스는 운에 가까움." Score 거울: "매수 5건 중 2건만 리스크 사전 기입." 민수가 자기 패턴을 본다.

실패: 시세 API 지연 → 틀린 값을 조용히 보여주지 않고 as-of 명시, 누락 지표 경고. 복기는 본 정보로만 평가.

### Flow 2 — 민수(Primary) 엣지: 공포 매도 직전 직면

1. 시장 급락, 보유 종목 -8%. 민수의 손이 매도 버튼으로 간다.
2. 매도 전 체크(매도 이유?·감정?, 탭 선택). 감정 '공포'.
3. 앱이 *그가 매수 시 적었던* "투자 기간 6개월 / 손절 -15%"를 들이민다(FR39) — "현재 -8%, 당신의 손절선은 아직."
4. **클라이맥스:** 민수가 과거 자기 논리와 **직면**해 공포 매도를 멈춘다. 막은 게 아니라 비췄다.
5. (매도하더라도 그 결정은 기록되어 매수 결정과 연결되고 다음 복기 대상이 된다, FR40.)

실패: 시세 오류 → "as of 16:00 EOD · 실시간 아님" 명시, 누락 지표 가시 경고. 복기 공정성 유지.

### Flow J3 — 지영(Secondary): 출근길 뉴스 → Daily Brief

1. 출근길, "금리 인하 기대 기술주 급등" 헤드라인 — 지영이 FOMO를 느낀다.
2. Daily Brief를 연다 — 일반 요약이 아니라 *"이 뉴스는 당신 관심종목 X·Y에 긍정적일 수 있으나 이미 선반영 → 변동성↑"*(관심종목 영향, FR29).
3. 충동 진입 대신 리서치 카드로 가격 구간과 실적일을 확인한다.
4. **클라이맥스:** 5분 안에 시장+내 종목 상태를 파악하고 **충동을 넘긴다**. 뉴스 과민 반응 루프를 끊는다.

### Flow J5 — Lurker → 첫 기록 전환

1. 2주간 Daily Brief·리서치 카드만 보고 기록은 0(가장 흔한 패턴).
2. 관심종목이 급등, "살까" 고민하며 리서치 카드를 연다.
3. 카드 하단 *비강제* 넛지 — "이 결심, 30초로 남겨두면 한 달 뒤 옳았는지 알려드려요"(가치 약속, 강요 없음, FR31).
4. **클라이맥스:** 첫 결정 기록 → 판단 루프 진입. *획득(리서치) → 리텐션(판단)* 다리를 건넌다.

## 미해결/갭

자체 점검: 위 IA 표의 모든 표면이 Voice/Component/State/Primitive 섹션 참조 + 최소 1개 Flow 또는 Pattern으로 커버됨을 확인.

- **운영자(최소, FR36–38):** IA·State에 명시했으나 별도 Key Flow가 없다(PRD J4 운영 여정 존재). MVP에서 사용자향 플로우가 아니라 의도된 축소 — `[NOTE FOR UX]` 운영 화면은 별도 내부 스펙으로 다룰지 확인 필요.
- **Daily Brief 트리거:** PRD상 MVP는 on-demand(cron 자동화 아님). 본 문서는 표면만 정의 — `[ASSUMPTION]` 사용자가 수동으로 여는 것으로 가정(자동 푸시 없음).
- **로그인/가입(FR1–2):** IA에 있으나 Key Flow 없음 — 표준 인증으로 간주, 별도 흐름 불요로 판단. 확인 필요 시 `[NOTE FOR UX]`.
- **결정 이력/물타기 표면(FR15–16):** State(append-only/버전)와 IA로 커버되나 전용 Flow는 Flow 2의 매도-매수 연결에 종속 — 독립 탐색 플로우는 Growth 단계로 미룸(`[ASSUMPTION]`).
